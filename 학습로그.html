[JDK] 람다의 변수 범위 - Effectively final, final

내용

리팩토링 전, 기능 구현을 할 때 for 문 내에 stream을 쓰게 되었는데, 반복문의 i 를 람다식 내에서 사용하려니 컴파일 에러가 났다.
“Variable used in lambda expression should be final or effectively final”

리팩토링 전 live한 코드 🥲

람다식 외부에서 선언된 i 는 값이 변할 수 있어 신뢰할 수 없으므로 컴파일 에러가 발생한다.

i 를 index 에 할당하였다.
람다식은 외부 변수 index 를 사용하고 있지만, 람다식이 사용되는 블럭 내에서 값이 변하지 않기 때문에 effectively final 이다.

람다식은 외부 block에 있는 변수에도 접근할 수 있지만 해당 변수가 불변으로 선언되었을 때만 가능하다.
불변으로 선언해야 하는 이유는 지역 변수는 스택에 저장되기 때문에 람다식에서 값을 바로 참조하는 것에 제약이 있어
복사된 값을 이용하게 되는데, 이때 멀티 쓰레드 환경에서 복사 될/복사된 값이 변경 가능 할 경우 이로 인한 동시성 이슈를 대응할 수 없기 때문이다.

java 8부터는 final로 선언되지 않아도 초기화 된 이후 값이 한번도 변경되지 않았다면
자바 8에서는 final 키워드를 붙이지 않아도 붙힌 것과 동일하게 컴파일러에서 처리하며 이를 effectively final이라고 할 수 있다.

[JDK] Map vs FlatMap

내용

하나의 입력을 다시 stream으로 변환을 하고자할때 stream을 두번 쓰지 않고 flatMap() 사용

map()

단일 스트림 안의 요소를 원하는 특정 형태로 변환할 수 있다.
파라미터로 받은 mapper에서 extends R를 반환한다.

<R> Stream<R> map(Function<? super T, ? extends R> mapper);

flatMap()

flatMap은 map과는 다르게 mapper의 실행결과를 또다른 스트림으로 리턴한다.
stream의 값들이 mapper을 돌때마다 새로운 스트림이 생성되는데, 이들을 나중에 합쳐서 하나의 스트림(Stream<? extends R>)으로 반환한다.

<R> Stream<R> flatMap(Function<? super T, ? extends Stream<? extends R>> mapper);
