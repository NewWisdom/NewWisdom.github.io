지하철 미션을 코틀린으로 구현해보기

Data class는 상속이 안된다.

Custom Exception을 data class로 만들고 싶었다…. 하지만 실패

  • 왜냐하면 equals() 메소드를 제대로 오버라이딩해서 구현할 수 있는 방법이 없기 때문
  • ex) A Data Class는 자동적으로 equlas를 구현하는데, 이를 상속받은 B Data Class 또한 자동적으로 equlas를 구현하면 충돌 발생

Custom Exception은 일반 class로 만들더라…

Open

  • 코틀린에서는 모든 클래스가 기본으로 final
  • 때문에 상속을 허용하려면 open 키워드를 붙여주어야 함
open class BusinessException(val httpStatus: HttpStatus, val errorMessage: ErrorMessage) :
    RuntimeException() {
    constructor(httpStatus: HttpStatus, message: String) : this(httpStatus, ErrorMessage(message))
}

build.gradle에 이거 추가해줘야함

allOpen {
    annotation("javax.persistence.Entity")
    annotation("javax.persistence.MappedSuperclass")
    annotation("javax.persistence.Embeddable")
}
  • Entity가 되기 위해서는 클래스가 final이면 안된다.
  • 그런데 코틀린은 모든 클래스가 final이니… entity 클래스들에 open을 일일히 붙여주면 될까 ?
  • build.gradle을 통해 @Entity 어노테이션이 있으면 open 클래스로 만들 수 있다.
  • 참고 - Spring Boot + Kotlin + JPA 적용하기 Entity 생성시 생각해볼 점들
  • Kotlin JPA Entity 모델링하기

일급 컬렉션

@Entity에서 일급 컬렉션을 어떻게 쓰냐?

@Entity
class Line(
    @Id
    @GeneratedValue(strategy= GenerationType.AUTO)
    @Column(name = "LINE_ID")
    val id : Long = 0L,
    @Column(unique = true, nullable = false)
    val name : String,
    @Column(unique = true, nullable = false)
    val color : String,

    val extraFare : Int? = null,

    @Embedded
    val sections : Sections? = null
)
  • 일급 컬렉션을 가진 필드에 @Embedded 를 붙여줌
@Embeddable
class Sections(
    @OneToMany(mappedBy = "line")
    val sections: MutableList<Section>? = null
)

  • 일급 컬렉션에는 @Embeddable 붙여줌

  • 둘 다 명시해줘도 되고, 둘 중 하나만 선언해주어도 동작
  • @Embeddable은 실제로는 하나의 객체지만, DB의 테이블에서는 해당 객체 내부에 있는 필드들이 풀어져서 만들어짐

Update

  • 굳이 update() 를 만들지 않아도, 다음과 같이 엔티티를 수정하고 save() 하면 자동으로 update 쿼리가 나간다.
fun update(id: Long, lineRequest: LineRequest) {
    val updateLine = lineRepository.findLineById(id) ?: throw LineNotExistException()
    updateLine.update(lineRequest)
    lineRepository.save(updateLine)
}

save 하다가 sql 예외가 발생하면(나는 name의 유니크가 깨지는 요청) autoIncrement는 증가한 상태가 됨

@JvmStatic

  • 동반 객체에 정의 된 함수에 대한 정적 메서드를 생성할 수 있음
  • 어노테이션을 붙인 요소가 속성이면, static getter/setter를 만들어줌
  • 동반 객체내에 있는 메서드를 사용하려면 해당 어노테이션을 붙여주어야 함
